# Vue `ref` vs `shallowRef` 速查表


| 特性           | `ref`                              | `shallowRef`                          |
| -------------- | ---------------------------------- | ------------------------------------- |
| 响应式深度     | 递归代理所有嵌套属性               | 仅追踪`.value` 的引用变化             |
| 适用对象大小   | 小型、普通对象 / 数组              | 巨型对象、第三方库实例、DOM、WebGL 等 |
| 性能开销       | 首次渲染时遍历全部属性，可能爆栈   | 极低，仅监听引用                      |
| 循环引用       | 会无限递归 →`RangeError`          | 不受影响                              |
| 手动触发更新   | 不需要                             | 需要`triggerRef(ref)`                 |
| 典型错误       | `Maximum call stack size exceeded` | 视图不更新                            |
| 修复视图不更新 | —                                 | `triggerRef(ref)`                     |
| 与 Pinia 配合  | 避免把大对象放入`state`            | 用`shallowReactive([])` 存列表        |

## 常见场景


| 场景                      | 推荐写法              |
| ------------------------- | --------------------- |
| ArcGIS`MapView` / `Layer` | `shallowRef(view)`    |
| 图层列表                  | `shallowReactive([])` |
| 简单计数器                | `ref(0)`              |
| 需要深层响应的表单        | `reactive({ ... })`   |

## 快速记忆

> **大对象用 `shallowRef`，小对象用 `ref`，循环引用必用 `shallow`**。
